GA & GTM 使用心得(一) 總集篇

Nov 04 2019

目的

用GA( Google Analytic ) 做量化研究的目的，是觀察統計資料、比較趨勢；

如果要看單一使用者的使用情況，我還是會依靠當面的Usability Test或hotjar的session record。

GA的基礎應用，是追蹤全網站各頁的PageView資料。

進一步可看：

① 各頁的bounce rate

② Landing Page & Leaving Page

③ 1/7/30天的回訪率

④ 流量來源


但若要做更細緻的資料，便需要 RD 協助埋各種不一樣的追蹤碼、或是透過 GTM ( Google Tag Manager )來埋特定的追蹤碼，後者也正是我使用的方法。

以下是2017~2019年間，由淺入深的過程中陸續克服的課題。

  1. User ID
    1. User ID追蹤 ( 原因、難處 )
    2. GA上對 User ID 做進一步辨識 ( IIP Viewer & Custom Segment )
  2. Event追蹤
    1. Event欄位分配
      1. 以按鈕為例，透過Variable和Click Text來回傳以精簡的Tag來追蹤大量Button
      2. Regular Expression 幫助寫出更精簡的Trigger
      3. 閱讀百分比 ( in SPA 追蹤 )
      4. 在SPA(Single Page Application)追蹤時的應做的變通

  3. 其他層面的問題

    1. 創造需求：我如何引導公司逐漸重視專案的GA量化指標，相對應的工作流程
    2. GTM快速Debug
    3. GAIQ證照考取
